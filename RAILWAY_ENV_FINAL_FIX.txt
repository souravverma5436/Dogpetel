========================================
WHAT WAS THE ISSUE?
========================================

Railway was copying the local server/.env file during deployment.
That file had:
  DB_HOST=localhost
  DB_NAME=petel_db

These values were overriding Railway's environment variables.

By adding .env to .railwayignore, Railway will no longer copy
the .env file, and will use the environment variables you set
in the Railway dashboard instead.
